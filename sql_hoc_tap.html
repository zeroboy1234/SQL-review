HAVING – lọc sau GROUP BYBắt buộc▼

Chỉ lấy phòng ban có từ 2 người trở lên. WHERE không dùng được với aggregate — phải dùng HAVING.

SELECT department, COUNT(*) AS so_nguoi FROM users GROUP BY department HAVING COUNT(*) >= 2;

Primary Key (PK)Bắt buộc▼

Primary Key là cột định danh duy nhất mỗi dòng. Không được trùng, không được NULL. Thường là cột id.

id (PK)nameemail
1Anan@gmail.com
2Bìnhbinh@gmail.com
3Chichi@gmail.com
CREATE TABLE users ( id SERIAL PRIMARY KEY, -- tự tăng, không trùng name TEXT, email TEXT );

Trong Odoo, mọi bảng đều có id SERIAL PRIMARY KEY được tạo tự động.

Foreign Key (FK)Bắt buộc▼

Foreign Key là cột ở bảng này trỏ tới Primary Key của bảng khác. Đây là lý do JOIN hoạt động được.

Bảng users

id (PK)name
1An
2Bình
3Chi

Bảng orders

id (PK)productuser_id (FK)
101Bàn phím1
102Chuột1
103Màn hình2
104Tai nghe3

user_id = 1 nghĩa là "đơn hàng này thuộc về người có id=1 trong bảng users". Đây chính là điều kiện ON trong JOIN.

CREATE TABLE orders ( id SERIAL PRIMARY KEY, product TEXT, user_id INTEGER REFERENCES users(id) -- FK );
One-to-Many (1 → N) — phổ biến nhấtBắt buộc▼

1 khách hàng có thể có nhiều đơn hàng. Trong Odoo đây là Many2one field.

1 user
→→→
nhiều orders
Dùng FK: orders.user_id → users.id
One-to-One (1 → 1)Nên biết▼

1 user có đúng 1 tài khoản hệ thống. Ít gặp hơn One-to-Many.

→
1 profile
Dùng FK + UNIQUE constraint

Không thể dùng FK thông thường. Cần tạo thêm 1 bảng trung gian chứa 2 FK. Odoo tự tạo bảng này khi dùng Many2many field.

nhiều students
→
enrollments

course_id (FK)

nhiều courses
NULL vs 0 vs chuỗi rỗngBắt buộc▼

NULL không phải số 0, không phải "". NULL nghĩa là "không có giá trị" — hoàn toàn vắng mặt.

Giá trịÝ nghĩaVí dụCách so sánh
NULLKhông có giá trịChưa nhập SĐTIS NULL
0Có giá trị, bằng 0Tổng tiền = 0đ= 0
''Có giá trị, chuỗi rỗngTên = ""= ''
-- Đúng SELECT * FROM users WHERE phone IS NULL; SELECT * FROM users WHERE phone IS NOT NULL; -- Sai: luôn trả về rỗng vì NULL không bằng gì cả SELECT * FROM users WHERE phone = NULL;
DBeaver & pgAdminBắt buộc▼

Cần 1 công cụ để kết nối PostgreSQL và chạy thử SQL.

DBeaver
Miễn phí, hỗ trợ mọi database. Giao diện trực quan, dễ xem cấu trúc bảng. Khuyên dùng cho người mới.
Khuyên dùng
pgAdmin
Công cụ chính thức của PostgreSQL. Đầy đủ tính năng hơn nhưng giao diện phức tạp hơn một chút.

Các bước kết nối (DBeaver)

1. Tải DBeaver tại dbeaver.io
2. New Connection → chọn PostgreSQL
3. Điền: Host = localhost, Port = 5432
4. Database, Username, Password do Odoo setup sẵn
5. Bấm Test Connection → Finish
